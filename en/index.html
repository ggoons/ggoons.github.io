Introduction

GGoons is a coding agency service. If you send specification of the each page of system that you want to develop to ggoons, ggoons will provide link to download backend and frontend source code that implements this specification. The source code provided by ggoons is not the best quality yet, but we assume that there may be some users who need it. So we decide to start the service.

Customers

GGoons generate source code that handles data input / output by user request. The source code provided by ggoons is only focused on implementing the functionality of the user's written statement, not the finished system that is available to end users targeting the sophisticated UI. Therefore, the users of the service are developers who can maintain the source code provided by the ggoons.

How can I use

  • If the user sends a code generation request along with the system statement to ggoon@ggoons.com, a code generation contract will be initiated.
  • In the first sentence of the email, "I have read and fully understood the usage and pricing policies of the home page, and I am requesting the code generation.".
    Requests that do not begin with this statement are ignored.
  • Then specify the backend type and the frontend type. (One for each device.)
  • And specify the DBMS type of the backend.
  • And specify company name and project name. (If present) These names are used in the package name of the source code.
  • And compress the system specification into a zip file and add it as an attachment.
  • If you have written your handwriting, please take a picture and add it to the zip file.
  • Sample: 
    I have read and fully understood the usage and pricing policies of the home page,
and I am requesting the code generation.

backend: express with typescript
frontend: angular, ionic
DBMS: MySQL
company: ggoons
project: coding

Attachment: xxx

  • At this point, ggoons will analyze the statement and notify the price to you.
  • The user must deposit 10% of the price notified by the ggoons to contract fee.
  • Once the deposit has been confirmed, you will be notified of the executable address after generating the code.
  • The user reviews the completed run screen and decides whether to purchase it.
  • If you are not satisfied with the results, you can decline the purchase. In this case, the down payment is not returned.
  • If you decide to make a purchase, deposit the remaining balance.
  • When the deposits are confirmed, ggoons send the full source code to the user, and the contract ends.

  • Once the transaction is over, basically all the materials will be destroyed.
  • However, if the user wants to append or delete the table or column after this, it can be specified as the target of modification.
  • If the project is subject to modification, the statement will be retained, and subsequent revision requests will only the number of columns will be charged.
  • If not, the statement and any generated code will be discarded on the ggoons server, and subsequent modifications additional charges are incurred for all columns in the submitted statement.

  • The modifications mentioned here mean modifications of the specification.
  • That is, it does not imply modification of the generated source code. The source code is always generated based on the specification.
  • Therefore, once you have modified the generated source code, you must modify the source code to match the new specification if you have new one.

Reference

There is no format in the statement. Freely, each screen(hereinafter referred to as a page) belonging to the system you wish to develop define what data should be input / output. Here are some caveats.

  • Please write the specification on a page-by-page basis.
  • Please specify what data should be input / output to the page.
  • Ignored except for the data I / O function specification. (eg: button color, font size, etc.)
  • External systems are supported only if they are accessible via apikey or OAuth 2.0.
  • If you are using social login or external system data, you must provide a schema for that external system.
    Here, schema refers to the address for accessing the external system and the request / response data structure of the external system.

  • If a database schema is provided:
    • The schema must be provided as an SQL file for MySQL.
    • The schema must have sufficient integrity to implement the specification.
    • When defining a page function, use the table name and column name.
    • If the map should display on the page, the longitude / latitude or address information must be present in the table.
    • If you have a table that needs state management, add the table name to the state table.
      ex) state table: tabl1, table2, ...
    • If there is a state management table, select the state management library. ( ngrx or ngxs )
    • The source of the data used on the page must be clear. (Table or external system)

  • Statistics and charts are only allowed if the DB being used is an RDB.
  • Specify indent of the source code. (TAB or n space)
  • If there is a CRUD logic to execute when a specific event occurs, such as a button or a batch operation, Write the logic in pseudo code.

Price

We have two kind of price policies.

Project base

This is pure contract base system. Customer and ggoons will meet in some place and writing contract to start project. In this case, price for ggoons service is 20% (without tax) of the customer's peoject budget. After contract, from the beginning of the project until the design is complete, ggoons will provides an unlimited number of implementations of the design. And also provides a detailed description of the source code generated so far to the UI enhancement developer(s).

Column base

The price is only depends on number of columns of the each pages in specification. So, price = (number of columns in all pages) * (column price). And the column price is determined as follows:

Database schema Column price Misc
Not provide 22$ (USD)
Provide 2$ (USD)

Because there is a large difference in column price, you should be more careful whether database schema is provided or not. In addition, if there is an error in the supplied specification or database schema, you should be aware that the contract may be terminated depending on the extent of the error. Smaller errors will be adjusted, but if there are too many errors, the project may not be able to continue. In this case, the down payment deposited by the user will not be returned.
